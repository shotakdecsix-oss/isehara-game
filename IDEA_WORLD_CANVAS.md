# アイデアメモ:World Canvas(仮題)

**ステータス**:構想メモのみ。実装なし。

**一行コンセプト**
地形・線路・道路だけを残したまっさらな世界に、不特定多数のプレイヤーが自由に建物や絵を置いていき、みんなの創作物だけで街が出来上がっていく共創サンドボックス。GPS連動でスマホカメラ越しの現実に重ねて見られる。

**ターゲット・体験の軸(2026-07-15追記)**
子供向け。LEGO/マイクラのように自分でブロックを組み立てて「おもちゃ」を作り、それを現実の街の好きな場所に飾る。その場所に実際に行けば、そのおもちゃとARで一緒に写真が撮れる。地域は限定しない(グローバル前提)。GPS精度は厳密でなくてよい(建物レベルの正確な重畳は不要、近づいたら出てくる程度で十分)。

**拡張:ギャラリー兼アルバム構想(2026-07-15追記2)**
3D空間を、子供も含むユーザーのSNSの場にするイメージ。3つの創作/収集の系統がある。
1. ゲーム内で作ったおもちゃを街に飾る(上記)。
2. 現実で描いた絵・作った作品を写真から3Dオブジェクト化して街に飾る。
3. 自分が持っている写真の位置情報をもとに3D空間にマッピングし、世界を移動しながらその場所であった思い出を振り返れる。

ゲーム内の3D空間が「ギャラリー」であり「アルバム」でもある、という統一メタファー。特に3番目(写真の位置マッピング)は、他のプレイヤーが誰もいなくても自分一人の思い出アルバムとして初日から完結する価値があり、「不特定多数が集まらないと街に見えない」という当初の弱点を解消する核になりうる。

## ベースとなる世界
現状の実装は「世界を3D空間として移動できる」機能のみ(文明育成シムはDESIGN_NOTE.md止まりの構想で未実装)。
その3D移動基盤の延長線上に、専用の1ワールドを新設する想定。
地形(標高・河川・海岸)と線路・道路網だけを引き継ぎ、建物・史跡・文明要素は一切なしの状態からスタート。

## コア体験
- プレイヤーはマップ内を自由に歩き回る。
- 好きな場所に自分で建物を建てたり、絵を描いたりできる(具体的な作成手段=ビルドツールか自由配置かは未定)。
- 特定のプレイヤーの育成対象ではなく、不特定多数が同じ世界に作品を置き続けることで街が有機的に育っていく。誰か一人の「街」ではなく、参加者全員の集合作品。

## GPS連動 VR/AR 機能
- 実世界での現在地・移動先とゲーム内座標を連動。
- スマホカメラに映る現実の風景に、その場所にプレイヤーたちが作った建物・絵をAR的に重ねて表示。
- つまり「ゲーム内の仮想の街」と「現実の位置情報」がひも付き、実際に出歩くと自分たちの作った街が現実の上に見える。

## 実現性の見立て(2026-07-15、対象=子供向けと判明後)
- **AR/GPS精度**:精密な建物レベル重畳は不要になったため、実現難度は当初想定より低い。近接検知程度のラフなAR表示で体験として成立する。
- **街としての密度**:不特定多数が同じ場所に集まる前提ではなく、各子供が自分の思い出の場所(自宅前・通学路・公園)に個人的に置く体験が核。密度がなくても一人ひとりの体験として成立するため、地域無制限でも過疎感は致命的ではない。
- **最大の論点=子供の位置情報の安全性**:「このおもちゃはここにある」を誰でも見られる設計は、子供の実際の居場所を第三者に晒すリスクに直結する。デフォルト非公開(家族・承認済み友人のみ)、公開する場合も位置を大まかにぼかす、といった設計が前提になりそう。COPPA/GDPR-K等、子供のデータ・位置情報を扱う法規制も要確認。
- **創作手段は自由描画よりブロック組み立てが安全側**:自由描画は内容モデレーションがほぼ不可能。LEGO/マイクラ的なパーツ組み立てなら語彙が限られ、不適切コンテンツの自動チェックもしやすい。
- **ストア審査**:子供向けアプリはGoogle Play Families / Appleキッズカテゴリ等、年齢確認・保護者同意フローの要求が重い。後回しでよいが認識しておく。
- **3要素で実現難度・リスクが大きく異なる**:①おもちゃ作成(中程度、公開前提でモデレーション要)②写真→3Dオブジェクト化(平面の絵は板状オブジェクト化で今すぐ着手可、立体の工作物は複数角度撮影かLiDARが要り本格的なフォトグラメトリは難易度高)③写真の位置マッピングによる思い出アルバム(Exif位置情報を読むだけで実現でき、技術難度は最も低く、非公開前提なら安全性の懸念も小さい)。
- **着手順の示唆**:③(個人アルバム)は多人数が揃わなくても単独で価値が完結し、既存の3D移動基盤をそのまま使えるため、最初に育てる核として筋が良い。①②はSNS的な公開を伴う分、公開範囲設計・モデレーションを先に固めてから着手が安全。
- **プライバシー設計は二層構造になりそう**:個人のアルバム層(写真+位置、デフォルト非公開・自分/家族限定)と、街づくりの公開層(おもちゃ等、明示的に公開を選んだものだけ)。写真+位置+子供、という組み合わせは特にセンシティブなため、アルバム層は既定で非公開を強く維持する。

## 類似サービス調査(2026-07-15)
- **Minecraft Earth**(Microsoft/Mojang, 2019-2021):まさに「作った建物をARで現実に重ねて、外を歩いて他プレイヤーと協力構築」という本構想と同じコンセプトを最大手が実施したが2021年6月に終了。公式理由はコロナ禍で屋外協力プレイが成立しなくなったこと。コンセプト自体の否定というより外出前提設計がパンデミックに弱かった可能性はあるが、大手が本気でやって畳んだ事実は重い。
- **Niantic Lightship VPS**:現実の場所にセンチメートル単位でAR物体を永続固定し複数ユーザーに同じものを見せる基盤technology。世界125都市14万拠点以上でサードパーティに開放済み。Minecraft Earth時代に一番難しかった「正確な現実重畳」を自前で作らずに乗れる可能性がある。
- **LEGO Fortnite**(Epic Games×LEGO, 2023年12月〜):現実位置とは紐づかない純粋仮想空間の建築ゲームだが、ローンチ時同時接続240万人。子供向け・安全性重視の設計でも大ヒットが成立することを実証。「おもちゃ作成」要素の需要面の裏付けになる。
- **Memory Map / MemoMap / Memory Anchor Explorer / Globetrotter / Google Photos地図表示**:写真や思い出を現実の場所にピン留めして振り返る・他人の思い出を探索する、という体験は既に複数アプリがカテゴリとして存在し、需要は実証済み。ただし調査範囲では全て上から見る2Dマップ型で、一人称視点で歩き回れる3D空間として作っている例は見当たらなかった。
- **総合判断**:個々の要素(子供向けUGCビルディングの需要、AR基盤の実用性、写真×位置の振り返り需要)はそれぞれ他者がすでに実証済み。ただしそれを「歩いて巡れる3D空間」として統合した例はまだない。全体を一気に作るとMinecraft Earthと同じ轍を踏むリスクがあるため、需要実証済み かつ 3D化で誰もやっていないニッチである③写真の位置マッピング(3D思い出アルバム)から着手し、段階的に②→①へ広げるのが妥当という結論に至った。

## 親和性の高い追加アイデア(2026-07-15)と優先度確定
③写真の位置マッピング(思い出アルバム)を核に、以下を追加検討。ユーザーが特に良いと判断したのは**成長の年輪(定点観測)**、**世代・家族をまたぐ広がり**、**歩数・訪問回数によるゆるいインセンティブ**の3つ。
- **成長の年輪(定点観測)**:同じ場所(桜の木、校門、神社など)で毎年同じ日に撮ることで、訪れるたびに過去の自分やおもちゃが時系列に積み重なって見える。繰り返し訪れる動機になる。
- **世代・家族をまたぐ広がり**:家族旅行のアルバムを家族で持ち寄って後で一緒に追体験。祖父母世代の古い写真を重ねて「昔はここに何があったか」を孫が知る、世代を超えた記憶の継承にもなる。二層プライバシー構造(個人非公開/家族友人限定)と相性が良い。
- **歩数・訪問回数のインセンティブ**:Pikmin Bloomのような、罰のない「歩くと良いことがある」設計で、思い出を残すために外に出る動機を作る。
- (検討したが優先度低:タイムカプセル機能、音声/動画添付、地域振興・観光連携)

**地域スコープの方針転換**:当初は伊勢原(本プロジェクトの舞台)との連携を想定していたが、ユーザーはもはや伊勢原という特定地域にはこだわっていない。地域限定の観光・自治体連携アイデアは前提から外し、グローバル・場所非依存の設計で進める。
