# ChronoDrift ローカルサーバ (プロトタイプ)

標高API (opentopodata) と OSM Overpass API をサーバ側でプロキシ+ディスクキャッシュし、
2回目以降の地形・マップ読み込みを即時にします。**index.html は一切変更していません。**

## 起動

```
node server/server.js
```

ポートを変えたい場合: `node server/server.js 3000` または環境変数 `PORT`。

## アクセス

- PC: http://localhost:8080/
- スマホ (同じLAN): http://192.168.0.14:8080/ (従来どおり。0.0.0.0 でリッスンしています)

※ これまで使っていた静的サーバ (8080番) は止めてから起動してください。

## 仕組み

- サーバが index.html を配信する際、`fetch` をプロキシ (`/api/elevation`, `/api/overpass`) に
  向ける小さなスクリプトを注入します。ファイル自体は無変更です。
- 初回アクセス: 上流APIへ 1.1 秒間隔でリクエスト (レート制限厳守) し、結果を
  `server/cache/` に保存します。初回は従来と同程度の時間がかかります。
- 2回目以降: キャッシュから即応答。さらにキャッシュヒット時はゲーム側の
  レート制限待ち (1.1秒/1.5秒) も自動短縮されるため、地形・タイルがほぼ一瞬で読み込まれます。
- 429 やタイムアウトはサーバ側で最大3回リトライするため、タイル歯抜けが起きにくくなります。

## キャッシュのクリア

`server/cache/` フォルダを削除するだけです (次回アクセス時に再取得)。
